# E10-S1-fernly-scaffold-recording-adapter ‚Äî Scaffold examples/fernly + in-memory recording adapter (headless no-op)

## Why

Every other E10 slice adopts through one shared harness: the invented product **Fernly** under `examples/fernly/`, wired to an in-memory recording `AnalyticsAdapter`. This story stands up that harness ‚Äî the workspace member, its type/test gates, the recording mock, and the unkeyed whole-stack no-op ‚Äî so the rest of the epic just adds config + assertions on top. This IS the bar-B substrate: a new app enrolled by config, running against a mock adapter, never a real backend.

## Scope

### In

- Create `examples/fernly/` as a **real pnpm workspace member**: add `examples/*` to `pnpm-workspace.yaml`; a `package.json` (`name: @example/fernly`, `private: true`) depending on the four packages via `workspace:*` (`analytics-kit`, `@analytics-kit/browser`, `@analytics-kit/node`, `@analytics-kit/react`) plus the React/jsdom devDeps S8 needs (`react`, `react-dom`, `@testing-library/react`, `@testing-library/dom`, `jsdom`, `@types/react`, `@types/react-dom`); a `tsconfig.json` extending `../../tsconfig.base.json` with `jsx: "react-jsx"` + `lib: ["ES2022","DOM"]`; a `vitest.config.ts` mirroring `packages/react/vitest.config.ts` (`environment: 'jsdom'`). Expose ONLY `typecheck` + `test` scripts ‚Äî **no `build`, no published output**. (`lint` optional; if added, no new machinery.)
- A **`RecordingAdapter`** in `examples/fernly/src/` implementing the seam's exported `AnalyticsAdapter` interface **in full** ‚Äî the mock must implement **every** member of the interface or the typecheck gate fails (`class RecordingAdapter implements AnalyticsAdapter`). It is an in-memory recorder that captures the write verbs into inspectable arrays, models a small anonymous‚Üíidentified identity state machine (see Technical notes; S3 exercises it), and returns benign values for the read/lifecycle verbs. The **exact** member set to implement (verified against `packages/analytics-kit/src/adapter.ts`, 18 members):
  - **Recorded write verbs:** `capture(event: NeutralEvent)`, `identify(distinctId, traits?, traitsOnce?)`, `register(props, options?)`, `unregister(key)`, `reset(options?)`, `group(type, key, traits?)`, `alias(previousId, distinctId)`, `setConsentState(state)`, `setPersistedProperty(key, value)`. (`alias` is a required member but is NEVER on the facade's identify/merge path ‚Äî record it inertly; the merge state machine lives in `identify`/`reset`/`getDistinctId`, not `alias`.)
  - **Benign read/lifecycle verbs (mirror `NoopAdapter`):** `getDistinctId(): string` (returns the modeled current id ‚Äî see state machine), `getConsentState(): ConsentState` (benign default, e.g. `'granted'` so the facade isn't opted-out-by-default and captures actually record ‚Äî see Technical notes), `flush(): Promise<void>`, `shutdown(): Promise<void>`, `fetch(url, options): Promise<NeutralFetchResponse>` (returns a `{status:0,‚Ä¶}` stub), `getPersistedProperty<T>(key): T | undefined` (`undefined`), `getLibraryId(): string`, `getLibraryVersion(): string`, `getCustomUserAgent(): string | undefined`.
- A `createFernlyAnalytics(...)`-style harness factory in `examples/fernly/src/` that adopts through the **seam** `createAnalytics(config, recordingAdapter, { generateUuid })` (the public second-param injection point) ‚Äî NOT the browser package's single-arg `createAnalytics`.
- A headless **no-op** proof: constructing the harness with `key` unset yields a whole-stack no-op that records nothing (bar B ‚Äî an unconfigured environment sends nothing). **The harness factory itself owns this branch** ‚Äî the SEAM `createAnalytics(config, adapter, deps)` does NOT gate on `config.key` (it uses whatever adapter is injected), so "unkeyed ‚áí inert" must be Fernly's OWN adoption decision: when `key` is unset, `createFernlyAnalytics` injects a `NoopAdapter` (imported from `analytics-kit`) ‚Äî or omits the recorder ‚Äî instead of the `RecordingAdapter`. This mirrors how the browser/node factories choose a real-vs-Noop adapter on key, kept on the consumer side here because the harness injects its own adapter. A vitest test asserts zero recorded calls after a `track` on the unkeyed harness.

## Technical notes

- **Harness shape = architect-ruled (2026-07-08), shape (A).** The whole harness adopts at the **seam** `createAnalytics(config, recordingAdapter, deps?)` ‚Äî VERIFIED: `packages/analytics-kit/src/create-analytics.ts` exposes exactly `createAnalytics(config: AnalyticsConfig, adapter?: AnalyticsAdapter, deps?: { generateUuid?: () => string })`, so injecting a mock as the second param is config-level adoption, not a library edit. The browser package's own `createAnalytics(config)` is single-arg (VERIFIED: `packages/browser/src/create-analytics.ts`) and builds a real `BrowserAdapter`; E10 deliberately does NOT use it, so a mock can be injected. NOTE the seam factory does `resolvedAdapter = adapter ?? new NoopAdapter()` and NEVER reads `config.key` ‚Äî key-gating is not a seam behavior (see the no-op Scope item). This is why "the browser slice is wired" (S8) is satisfied by the React binding + the neutral facade exercising the browser-relevant config surface (`cookieDomain`/`contexts`/`autocapture`), not by instantiating `BrowserAdapter`.
- **Why a mock adapter is the bar-B proof, not an integration test.** ‚Äî epic (2026-07-07): the harness runs against a mock/in-memory adapter, never a real backend, and THAT is what makes it bar B. The facade (`AnalyticsProviderImpl`) owns the allowlist gate, consent swap, `register`/super-props, `context()` scoped views, and `buildEvent` (dedupeId mint) ‚Äî all run against ANY adapter, so a recording mock exercises them faithfully.
- **RecordingAdapter identity state machine (feeds S3).** ‚Äî architect (2026-07-08): to keep the merge proof honest, `identify()` must model the three-branch neutral contract, not just store the last id: (1) new id while modeled-anonymous ‚Üí merge (retain prior anon id as the link, adopt new id, flip to identified); (2) same id ‚Üí traits-only, no re-merge; (3) new id while already-identified ‚Üí no client merge. `reset()` must re-anonymize (fresh anon id, drop the retained link, back to anonymous). `getDistinctId()` returns the current modeled id. This is ~15 lines of in-memory state ‚Äî it models the neutral contract, NOT `BrowserAdapter`'s cookie/`IdentityStore` internals (E4 owns those). Ship the state machine here so S3 can assert on it.
- **Interface to implement:** `AnalyticsAdapter` in `packages/analytics-kit/src/adapter.ts` (exported from `analytics-kit`). Read it for the exact 18-member verb set and the read-verb return contracts. Mirror `NoopAdapter` (`packages/analytics-kit/src/noop-adapter.ts`) for the inert reads and lifecycle verbs ‚Äî it is the canonical full-SPI implementation to copy the shape from (all 18 members, `fetch()` `{status:0,‚Ä¶}` stub, `getPersistedProperty` `undefined`, `getLibraryId/Version` strings, async `flush`/`shutdown`).
- **Consent-default footgun (load-bearing for S2‚ÄìS5 to bite).** ‚Äî verified against `AnalyticsProviderImpl` (`packages/analytics-kit/src/analytics-provider.ts` ~lines 108, 254‚Äì264): the facade resolves `optedOut` at construction from `adapter.getConsentState()`, and while opted out it swaps the ACTIVE delegate to an internal `NoopAdapter` ‚Äî so writes reach `noopAdapter`, NOT the injected recorder, and every downstream assertion sees an EMPTY stream. `NoopAdapter.getConsentState()` returns `'denied'`, and a `'pending'` state with no `consentDefault: 'granted'` also resolves to opted-out (fail-safe default). Therefore the `RecordingAdapter` must return a GRANTING consent state (`getConsentState(): ConsentState { return 'granted'; }`) ‚Äî OR the keyed harness config must set `consentDefault: 'granted'` ‚Äî so captures actually land on the recorder. Pin one of these in S1 and hold it across S2‚ÄìS8; without it the whole epic's stream assertions pass vacuously. (The unkeyed no-op proof below is the OPPOSITE case ‚Äî it relies on nothing being wired, so it records nothing regardless.)
- **Workspace/gate shape = architect-ruled (2026-07-08), shape (a).** Expose `typecheck` + `test`, omit `build`. VERIFIED `turbo.json`: `typecheck` and `test` both carry `dependsOn: ["^build"]`, so the example is checked against freshly-built package `dist` ‚Äî the exact surface an external npm consumer sees. The task definitions in `turbo.json` are global, so a new workspace member inherits them once it is enrolled in `pnpm-workspace.yaml` and exposes matching `package.json` scripts ‚Äî **no `turbo.json` edit is required** (and none should be made). VERIFIED `tsconfig.base.json` has NO `paths` aliases to any `src` ‚Äî so `examples/fernly` extending it resolves the four packages through node module resolution ‚Üí their `package.json` `exports`/`types` ‚Üí `dist/*.d.ts` (the built declarations, not `src`). Do NOT add any `paths`/alias that would short-circuit to `packages/*/src`, or the gate stops proving the external surface. VERIFIED `pnpm-workspace.yaml` today lists only `packages/*` ‚Äî S1 adds `examples/*`. Editing `pnpm-workspace.yaml` is repo-infra, NOT a bar-B violation (bar B scopes "library change" to `packages/*` source). The invariant to hold across all E10 stories: Fernly's own source edits zero `packages/*` files. **Ordering:** `turbo run typecheck` for E10 requires the four packages to build first ‚Äî S1 assumes E6/E7/E8/E9 are shipped and buildable (they are), so `^build` resolves.
- **Naming (locked, load-bearing):** Fernly is invented and neutral ‚Äî no real product/company. Fernly names appear ONLY under `examples/`; E11's name-scan is anchored to `packages/**` and explicitly excludes `examples/**`. Use a neutral package name (`@example/fernly` or similar) ‚Äî never bake a vendor name anywhere.
