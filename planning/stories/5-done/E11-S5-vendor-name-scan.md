# E11-S5-vendor-name-scan ‚Äî CI-able vendor/product-name scan (the critical gate)

## Why

The whole reason the library exists is that no vendor name leaks into its surface. This slice makes that a durable, re-runnable, exit-nonzero gate ‚Äî not a one-time grep ‚Äî so the neutrality guarantee holds as new code and adapters land. It ships FIRST because its confinement rule (describe-by-role, never by vendor) constrains what the docs stories (S1/S2) may write.

## Scope

### In

- A CI-runnable scan (a vitest test or a `scripts/` node script wired into a package's `test`/a root `turbo` task) that **exits nonzero on any match** across the library's SURFACE and passes on the correctly-shipped code.
- **Scan by DIMENSION, not by raw text** (architect-locked ‚Äî see Technical notes). A forbidden token is a FAILURE only when it lands in a consumer-observable dimension:
  - exported / public identifiers, type + interface names, enum members, public const names (checked against each package's single bundled declaration file ‚Äî `packages/*/dist/index.d.{ts,mts}`, BOTH the `.d.ts` and its `.d.mts` sibling; tsup rolls up ONE declaration bundle per package, there are no per-source `.d.ts` files ‚Äî see Technical notes. That bundle IS the public surface by construction);
  - `package.json` `name` fields + file/dir names under `packages/`;
  - consumer-reachable strings (thrown error messages, log output, JSDoc that lands in the published `.d.ts`);
  - every occurrence in a shipped doc (root `README.md` + the S1 matrix + the S2 guide ‚Äî docs have NO internal exemption).
- Forbidden tokens: case-insensitive `posthog`; `ph_` prefixes; region/vendor hostnames (e.g. `i.posthog.com`, `us.i.`, `eu.i.`); the invented product name `fernly`.
- **`$`-prefixed wire literals: gated by a CONFINEMENT RULE, not a whitelist** (architect-locked). A `$`-prefixed string LITERAL (not a `$`-token inside a comment ‚Äî the confinement rule is literal-scoped, so the `$`-tokens in the port-provenance comments pass untouched) is PERMITTED only as the value of a const whose exported identifier matches `/_WIRE_(EVENT|KEY)$/` (or lives in a `[WIRE]`-tagged region); it FAILS anywhere else in code (an escaped-confinement leak) and always FAILS in docs. The exported identifier itself is still scanned and must carry zero vendor token. **The confinement pass runs ONLY over `src` files that contribute to the shipped surface** (transitively reachable from the package's public `index.ts` entry) ‚Äî a `src` file NOT in the entry's import graph (test helpers, fixtures, harnesses) is out of scope regardless of filename (see Technical notes ‚Äî this is why `wire-scan.test-helper.ts`'s `'$insert_id'` literal does NOT trip the scan).
- **Exempt paths (skipped entirely):** `examples/**`, `planning/**`, `.claude/**`, `CLAUDE.md`, `posthog-js/**`. Test files (`*.test.ts`, `*.test-helper.ts`, fixtures) are out of scope for ALL dimensions BY CONSTRUCTION ‚Äî they never emit to `dist` (so they're absent from the identifier/type dimension), are not docs, and their strings aren't consumer-reachable; and for the `$`-confinement pass they're additionally excluded as unreachable-from-entry (see Technical notes). So a builder must NOT raw-scan test-file text for `posthog`/`fernly`/hostnames. Non-doc `//` comments in `packages/**/src` are skipped for the `posthog` token (port-provenance citations are dev-facing audit evidence, not surface ‚Äî see Technical notes).

### Out

- Fixing any hit the scan surfaces. Per the epic's audit-not-patch rule, a real leak routes to a bug against the epic that owns it ‚Äî E11 gates, it does not patch.
- Whitelisting individual `$`-literal values (rejected ‚Äî see Technical notes; the confinement RULE replaces the list).
- Scrubbing the port-citation comments from `packages/**` (rejected ‚Äî see Technical notes).

## Acceptance criteria

- [ ] The scan runs as an exit-nonzero CI check (a test or `turbo`/`scripts` task), not a manual grep; a planted `posthog` in a `dist` `.d.ts` identifier, a planted `fernly` in the README, or a `$pageview` moved OUT of a `_WIRE_` const each make it FAIL.
- [ ] On the current shipped tree the scan PASSES: the port-citation `// De-branded from posthog's ‚Ä¶` comments in `packages/**/src` (all 30 `posthog` hits in non-test src are `//` comments ‚Äî verified), the four `$`-literal `_WIRE_(EVENT|KEY)` consts in `browser/src/persistence-keys.ts`, AND the `'$insert_id'`/`'insert_id'` literals in the unreachable `browser/src/wire-scan.test-helper.ts` (a non-exported test helper, not in the entry import graph), do NOT trip it.
- [ ] The identifier/type/package dimensions are checked against the built `packages/*/dist/index.d.{ts,mts}` bundle ‚Äî BOTH the `.d.ts` and the `.d.mts` sibling (a consumer resolving the ESM condition reads `.d.mts`; scanning only `.d.ts` would miss half the declaration surface) ‚Äî not raw `src`, proving the neutral surface a consumer imports carries zero vendor token.
- [ ] The scan is documented so a future adapter passes the SAME gate with zero scan edits: a new wire token passes iff it obeys the `_WIRE_`-confinement convention (upholds the epic's "scan gains no exceptions").
- [ ] Bar-neutral proof: the exported identifier `PAGEVIEW_WIRE_EVENT` passes while its value `'$pageview'` is permitted only inside the confined const ‚Äî the vendor shape never crosses into the consumer's type space.

## Technical notes

Scan boundary ‚Äî architect-locked (2026-07-09):

- **Scan by dimension, not path.** "Surface" = what a consumer can observe. The BRIEF (¬ßZero vendor references, lines 27‚Äì28) enumerates the protected dimensions: public API, type names, package names, file names, docs, plus exported source identifiers (epic Note line 43). A match FAILS only in those dimensions. The cleanest implementation is to run the identifier/type/package match against the emitted declaration bundle (comments and non-exported internals simply aren't in it ‚Üí precise + cheap), and a strip-comments / AST (ts-morph) classification pass over `src` for the `$`-literal confinement rule. **A raw `grep` over `packages/**` is the wrong tool and would false-fail** on the categories below ‚Äî say so in the code so nobody reaches for it.
- **Declaration dimension = the tsup bundle, not per-source `.d.ts`** (verified against the real tree, architect-confirmed 2026-07-09). tsup rolls each package up into ONE declaration bundle ‚Äî `packages/*/dist/index.d.ts` PLUS its `packages/*/dist/index.d.mts` sibling (8 files total: 4 `.d.ts` + 4 `.d.mts`). There are NO per-source `.d.ts` files, so a builder must NOT walk `dist` expecting them. Glob `packages/*/dist/index.d.{ts,mts}` and scan BOTH extensions: they are distinct SHIPPED files (a consumer resolving the ESM condition reads `.d.mts`), so scanning only `.d.ts` silently skips half the declaration surface ‚Äî a correctness hole, not polish. On the current tree all 8 are token-clean (verified: zero `posthog`/`$`-literal/`insert_id` in any `dist/index.d.*`).
- **Port-citation `//` comments in `packages/**/src` are SKIPPED for `posthog`** (architect-locked): they are dev-facing provenance (`// De-branded from posthog's event-utils.ts:45-54`), identical in kind to the `planning/`/`CLAUDE.md` citations the epic already exempts. They are load-bearing audit evidence that the port was DE-BRANDED, not copied ‚Äî the very thing this epic reviews. Scrubbing them is REJECTED; leave an explicit "rejected, and why" line in the scan code so it doesn't resurface.
- **`$`-wire literals ‚Äî confinement RULE, not whitelist** (architect-locked): permitted ONLY as the value of a const whose exported identifier matches `/_WIRE_(EVENT|KEY)$/` (or a `[WIRE]`-tagged region). This is strictly STRONGER than a gate: it FAILS the build if a `$`-literal escapes that confinement (e.g. someone inlines `'$pageview'` in `browser-adapter.ts` instead of importing the const), so the scan actively defends the neutral seam rather than carving a hole in it. A per-value whitelist is REJECTED (it becomes a vendor-token registry shipped in-repo and violates "scan gains no exceptions"). The rule is **literal-scoped, not text-scoped**: a `$`-token appearing INSIDE a `//` comment (e.g. `De-branded from posthog's $session_entry_url`, `$prev_pageview_duration`, `$geoip_disable` ‚Äî all present in real port-citation comments) is NOT a string literal and passes untouched; only actual `$`-prefixed string literals in code are classified.
- **Confinement runs only over the shipped import graph, not by filename** (architect-locked 2026-07-09). The invariant is "does this file's code contribute to what SHIPS?" ‚Äî a `$`-literal that never reaches `dist` and is never consumer-reachable cannot leak. SPEC: the `$`-confinement AST pass runs only over `src` files transitively reachable from the package's public `index.ts` entry; a `src` file not in that import graph (test helpers, fixtures, harnesses) is out of scope for confinement regardless of its filename. This still FAILS a genuine escaped `$`-literal in real adapter code (any running adapter code IS reachable from the entry) and does NOT false-fail on the ONE real case: `browser/src/wire-scan.test-helper.ts:6` holds `'$insert_id'`/`'insert_id'` string literals but is imported only by test files, never from `browser/src/index.ts`, so it never ships to `dist` ‚Äî verified. Note "reachable from entry" is REAL import-graph reachability, NOT "re-exported from index" (an internal module deep in the entry's chain is reachable-and-shipped but not re-exported, and MUST stay in confinement scope). IMPLEMENTATION: walk the module graph from each package entry. FALLBACK (if a true graph walk is more than this gate should carry): a filename-based exemption enumerating the test-tooling patterns actually present under `src` ‚Äî `*.test.ts` AND `*.test-helper.ts` ‚Äî but if you take the fallback, label it in the code as an APPROXIMATION of the reachability invariant, not the invariant itself. Either way the current tree passes; the `*.test.ts`-only exemption as originally drafted was insufficient (it misses `*.test-helper.ts`).
- Prior art to build ALONGSIDE: `packages/browser/src/wire-scan.test-helper.ts` (`containsInsertId` ‚Äî a deep-shape scan the S5 scan can generalize). The confined `$`-literal consts are `PAGEVIEW_WIRE_EVENT` / `PAGELEAVE_WIRE_EVENT` / `AUTOCAPTURE_WIRE_EVENT` / `GEOIP_DISABLE_WIRE_KEY` in `browser/src/persistence-keys.ts` (lines 40‚Äì52); their exported names carry zero vendor token.
- Concrete match/skip matrix (from the architect consult): FAIL = exported identifiers/type names in `dist/index.d.{ts,mts}` ¬∑ package + file/dir names ¬∑ consumer-reachable strings (errors, logs, published JSDoc) ¬∑ ALL shipped docs ¬∑ a `$`-literal escaped from a `_WIRE_` const in a file reachable from the package entry. SKIP = non-doc `//` comments in `src` (incl. `$`-tokens inside them) ¬∑ `$`-literal inside a `_WIRE_` const/`[WIRE]` region ¬∑ `$`-literal in a `src` file NOT reachable from the package entry (test helpers/fixtures ‚Äî subsumes `*.test.ts` AND `*.test-helper.ts`) ¬∑ `examples/`, `planning/`, `.claude/`, `CLAUDE.md`, `posthog-js/`.
- This is a GATED CHECK (executable), not prose ‚Äî the epic's "executable-over-prose where possible" applies fully here; the scan is the whole point of the epic.
